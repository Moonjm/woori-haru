# 기념일 D-Day 앱 뱃지 설계

날짜: 2026-07-22
브랜치: `feat/dday-badge`

## 목적

기념일 관리에 등록된 기념일 중 하나를 선택하면, 그 날을 1일째로 세는 D+ 카운트를 앱 아이콘 뱃지에 표시한다. 앱을 열지 않아도 자정마다 숫자가 자동으로 올라간다.

## 요구사항

- 카운트 방식: 선택한 날짜가 1일째. 오늘 표시 숫자 = (선택일부터 오늘까지 경과일수) + 1.
  - 예: 2025-01-01 선택 → 2026-07-22 뱃지는 568 + 1 = 569.
- 미래 날짜를 선택한 경우: 그 날이 오기 전까지 뱃지 숨김(0), 당일부터 1.
- `recurring`(매년 반복) 여부와 무관하게 최초 날짜 기준 누적 카운트.
- 선택은 한 번에 하나만. 기기 로컬(UserDefaults)에 저장하며 커플 상대방과 공유하지 않는다.

## 아키텍처

### 선택 UI — `PairEventsView`

- 기념일 리스트 각 행에 뱃지 아이콘 버튼 추가.
- 탭 시 해당 기념일을 뱃지 대상으로 선택(기존 선택은 자동 해제), 선택된 항목을 다시 탭하면 해제.
- 선택된 항목은 아이콘 강조 표시로 구분.

### 신규 서비스 — `DDayBadgeService`

책임: 선택 상태 영속화 + 뱃지 즉시 갱신 + 자정 예약 알림 관리.

- UserDefaults 키에 선택된 기념일 id·날짜 저장(id: `Int`, 날짜: `YYYY-MM-DD` 문자열).
- `select(event:)` / `deselect()` / `refresh()` 제공.
- `refresh()` 동작:
  1. 오늘 숫자를 계산해 `UNUserNotificationCenter.setBadgeCount(_:)`로 즉시 반영.
  2. 기존 D-Day 예약 알림을 모두 취소 후, **앞으로 30일치** 자정(00:00) `UNCalendarNotificationTrigger` 알림을 예약. 각 알림은 제목·본문·소리 없이 `badge` 값만 설정 → 배너·소리 없이 뱃지만 조용히 갱신됨.
  3. 알림 식별자는 `dday-badge-` 접두사를 사용해 공부 타이머 알림과 분리. 취소도 이 접두사만 대상으로 한다.
- iOS 예약 알림 64개 한도 내에서 30일치만 사용해 공부 타이머 몫을 남긴다.

### 권한

- `NotificationScheduler.requestPermission()`의 옵션에 `.badge` 추가 (`[.alert, .sound, .badge]`).
- 뱃지 선택 시 권한 미허용 상태면 요청하고, 거부돼 있으면 "설정에서 알림을 허용해주세요" 안내 얼럿 + 설정 앱 이동 버튼.

### 갱신 트리거

- 앱이 포그라운드로 진입할 때(`scenePhase == .active`) `refresh()` 호출 → 30일 버퍼 리필. (하루기록 특성상 거의 매일 열리므로 실사용에서 끊기지 않음.)
- 선택/해제 시 즉시 `refresh()` 또는 전체 취소 + 뱃지 0.

## 엣지 케이스

- 선택한 기념일이 서버에서 삭제된 경우(상대방 삭제 포함): `PairEventsViewModel.loadEvents()` 결과에 선택 id가 없으면 자동 해제 + 예약 취소 + 뱃지 0.
- 로그아웃 시: 별도 처리 없음(선택 유지). 단, 재로그인 후 기념일 목록에 해당 id가 없으면 위 규칙으로 자동 해제된다.
- 날짜 계산은 `Calendar.current`(기기 로컬 타임존)의 자정 기준.

## 검증

- 프로젝트에 테스트 타깃이 없어(저장소 관례) 단위 테스트 대신 다음으로 검증한다:
  - 날짜 계산은 날짜를 파라미터로 받는 순수 함수로 분리해 로직을 고립.
  - `xcodebuild` 빌드 통과 확인.
  - 시뮬레이터에서 선택 → 뱃지 즉시 표시, 해제 → 뱃지 제거, 예약 알림 30개 등록 여부 수동 확인.

## 비범위

- 서버 푸시 기반 무한 갱신 (백엔드 작업 필요 — 추후 필요 시 별도 건).
- 홈 화면 위젯 D-Day 표시.
- 뱃지용 기념일 선택의 서버 저장/커플 공유.
